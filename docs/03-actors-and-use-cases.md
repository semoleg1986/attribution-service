# Акторы И Use Cases

## Акторы

- `Admin`: создает/управляет токенами и смотрит отчеты
- `Partner/Marketing`: использует каналы и кампании
- `Parent`: переходит по токен-ссылке и запрашивает доступ
- `Course Service`: валидирует токен и записывает конверсии

## Ключевые Сценарии

### Жизненный Цикл Токена
1. админ создает токен
2. токен распространяется по выбранному каналу
3. токен автоматически истекает по TTL

### Воронка Request/Paid
1. родитель кликает по реферальной ссылке
2. `attribution-service` записывает visit
3. `course_service` запрашивает resolve discount
4. `course_service` записывает `requested` conversion
5. админ в `course_service` подтверждает оплату
6. `course_service` записывает `paid` conversion

### Отчетность
1. админ фильтрует по каналу/токену/дате
2. сервис возвращает метрики click->requested->paid
3. сервис возвращает агрегаты revenue/discount
