# Инварианты И Политики

## Бизнес-Инварианты

1. `token` уникален.
2. discount policy токена неизменяема после первой paid-конверсии.
3. валидность токена требует `status=active` и `now < expires_at`.
4. TTL по умолчанию для токенов курсовой кампании равен `course.starts_at`.
5. paid-конверсия должна ссылаться на существующий requested-контекст.
6. upsert конверсии идемпотентен по внешней ссылке (`access_grant_id`, stage).

## Политики

- `AdminPolicy`: создание/отключение токенов и доступ к отчетам.
- `ResolvePolicy`: для невалидного/истекшего токена возвращает нулевую скидку.
- `ConversionPolicy`: только append-only факты, без разрушительных изменений.
