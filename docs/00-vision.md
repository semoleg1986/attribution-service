# Архитектурное Видение Attribution Service

## Назначение

`attribution-service` — bounded context для реферальных токенов, канальной атрибуции, правил скидок и аналитики конверсий.
Сервис покрывает полный funnel: click -> access request -> paid.

## Бизнес-ценность

Сервис обеспечивает:
- измеримую эффективность каналов по реальным paid-конверсиям
- token-based скидочный flow для запросов родителя на доступ
- централизованное управление скидками и жизненным циклом токенов
- прозрачный источник атрибуции для каждого оплаченного доступа

## Границы

### Входит в контекст
- `ReferralToken`
- `AttributionVisit` (факты кликов)
- `AttributionConversion` (факты request/paid)
- `DiscountPolicy`
- отчетность по каналам и токенам

### Не входит в контекст
- создание курсов и lifecycle enrollment (`course_service`)
- владение пользовательскими профилями (`users_service`)
- аутентификация и lifecycle токенов (`auth_service`)

## Принципы

- Contract-first API
- Неизменяемые attribution-факты
- Идемпотентные upsert операций конверсий
- Явные правила TTL и валидности токенов
